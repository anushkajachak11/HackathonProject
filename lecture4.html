Lecture 4: File Path in HTML and Boilerplate in HTML:

File Path System:

A website is not a single file. It's a collection of HTML, CSS, JS, images, videos etc. Browser needs the correct address to locate these files. This address is called a File Path.

Relative Path:

  • This path is based on the location of your current HTML file.
  • Used for files that are part of your own project.
  • Depends on folder structure.

Example:

<img src="photo.jpg">

Absolute Path:

  • This is the complete address of a file (starts from the domain name).
  • Does not depend on the HTML file's location.
  • Mostly used for external resources (Google Fonts, CDN, hosted images).

Example:

<img src="https://www.example.com/images/logo.png">

Windows vs macOS/Linux:

Feature Windows macOS / Linux
Root Drive Letter (C:\) Forward Slash (/)
Separator Backslash\ Forward Slash /
Example C:\Users\John\Docs\file.txt /Users/john/Docs/file.txt

HTML Boilerplate:

Boilerplate = standard template that sets up every HTML file properly. Without it, browser guesses → can cause quirks mode, broken characters, bad SEO.

Basic Structure:

<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Document</title>
</head>
<body>

</body>
</html>

Why Each Line:

  • <!DOCTYPE html>: tells browser to use HTML5 standards mode.
  • <html lang="en">: declares page language (important for SEO + screen readers).
  • <meta charset="UTF-8">: ensures correct text/characters everywhere.
  • <meta name="viewport">: makes site responsive on mobile.
  • <title>: text in browser tab + search results.
  • <body>: all visible content goes here.

Multipage Website:

Real websites use multiple pages (Home, About, Contact). All pages should share common header + footer and have navigation menu to connect them.

Navigation Example:

<header>
  <nav>
    <a href="index.html">Home</a>
    <a href="about.html">About</a>
    <a href="contact.html">Contact</a>
  </nav>
</header>

<div>, class, id:

  • <div>: generic box to group elements (block-level).
  • id: unique identifier → only 1 per page (like Aadhaar number).
  • class: reusable label → can be used on many elements (like Job Title).

Example:

<div id="main-header">Header Content</div>

<div id="featured" class="profile-card">Special Profile</div>
<div class="profile-card">Normal Profile</div>
<div class="profile-card">Normal Profile</div>

<span>:

Inline version of div. Used for highlighting specific words inside text without breaking the line.

<p>This is very <span class="highlight">important</span> text.</p>

Semantic Tags:

  • <header>: top section, usually contains logo + nav.
  • <footer>: bottom section, copyright, contact info.
  • <main>: unique main content of the page (only one per page).
